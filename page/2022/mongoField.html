mongodb project和addFields的妙用

2022-06-28

>说明数据准备场景分析总结

说明

本文仅介绍mongodb如何优雅的修改某字段为某值，重点project、addFields

project——mongodb聚合命令的参考与演示 (logerlink.github.io)

addFields——mongodb聚合命令的参考与演示 (logerlink.github.io)

数据准备

db.user.insertMany([
    {name:'Zhangsan',age:18,gender:'男'},
    {name:'Lisi',age:15,gender:'男'},
    {name:'Zhaomeng',age:16,gender:'女'},
    {name:'Xiaoxiao',age:18,gender:'女'},
])

场景分析

查询时，将name改成自我介绍的一句话。这个其实很简单，很快就能写出来了，但是我们可以发现其他字段并没有显示。

xxxxxxxxxx
db.user.aggregate([
    {$project:{
        name:{
            $concat:['大家好！我是','$name',',今年',{$toString:'$age'},'了']
        }
    }}
])

这个显示问题我们后面再处理，那假如我想将name隐藏，并输出成info呢？

xxxxxxxxxx
db.user.aggregate([
    {$project:{
        name:0,
        info:{
            $concat:['大家好！我是','$name',',今年',{$toString:'$age'},'了']
        }
    }}
])

按照字面意思，我们也很快的写出来了，但是运行结果怎么是错的？

这是因为mongodb的project管道在排除投影期间（name:0）不能包含字段或添加计算字段

xxxxxxxxxx
db.user.aggregate([
    {$addFields:{
        info:{
            $concat:['大家好！我是','$name',',今年',{$toString:'$age'},'了']
        }
    }},
    {$project:{
        name:0,
    }}
])

那现在我要将name改成自我介绍的一句话，其他字段也要一起显示出来，同样很简单

xxxxxxxxxx
db.user.aggregate([
    {$project:{
        name:{
            $concat:['大家好！我是','$name',',今年',{$toString:'$age'},'了']
        },
        age:1,
        gender:1
    }}
])

但不够优雅，如果我user表有很多字段呢？一个一个写下来吗？再改造一下

xxxxxxxxxx
db.user.aggregate([
    {$addFields:{
        name:{
            $concat:['大家好！我是','$name',',今年',{$toString:'$age'},'了']
        }
    }}
])

哎？怎么变成addFields的使用了？

总结

呃，其实就是project、addFields的结合妙用，回头看这个场景分析怎么越看越别扭，越理越抽象，和我想的好像不太一样.
